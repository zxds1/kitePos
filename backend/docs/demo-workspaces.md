# Demo Workspaces

The seeded demo shop catalog used by the mobile app lives in [`uza_pos/assets/demo_data`](/home/sugho/UZApoint/uza_pos/assets/demo_data).

## Purpose

Demo workspaces let a prospective owner or staff user inspect a realistic Storflo setup before committing real shop data. Each workspace includes:

- a category-specific shop profile
- seeded products
- seeded restocks
- seeded sales history
- category-matched local image assets

The current implementation is app-local by design:

- the app loads demo data from bundled assets
- the app writes demo products, sales, and restocks into local Hive boxes
- no backend seed import runs automatically for demos

## Contract

The JSON catalog is generated by [`tools/generate_demo_catalog.mjs`](/home/sugho/UZApoint/tools/generate_demo_catalog.mjs).

The generated file contains:

- `generated_at`
- `shop_count`
- `category_count`
- `shops`

Each shop entry includes:

- shop identity and current user
- location metadata
- `demo_summary`
- supplier metadata when applicable
- `products`
- `sales`
- `restocks`

## Operational Notes

- Demo sessions use a non-production local session token prefix and should not be treated as backend-authenticated sessions.
- Demo media is bundled as app assets, not uploaded to Cloudflare R2.
- If backend demo import is added later, it should remain isolated from production shops and media buckets.
